# 自动驾驶知识检索系统 - 用户指南

## 1. 快速开始

### 1.1 系统简介

自动驾驶知识检索系统是一个专门为自动驾驶研发团队设计的智能知识检索平台。系统可以帮助您快速查找传感器、算法、法规和测试相关的专业知识，并提供详细的文档引用。

### 1.2 主要功能

- **传感器查询**：查询摄像头、激光雷达、毫米波雷达、超声波雷达的技术参数和标定方法
- **算法查询**：查询感知、规划、控制算法的设计文档和技术细节
- **法规查询**：查询 GB/T 国家标准、ISO 26262 功能安全标准、测试规范
- **方案对比**：对比不同传感器方案或算法方案的优缺点
- **多文档推理**：汇总多个文档的信息，提供完整的技术方案

### 1.3 知识库范围

当前知识库 **ad_knowledge_v01** 包含：

- **传感器文档** (5 份)：摄像头、激光雷达、毫米波雷达、超声波雷达的规格书和标定文档
- **算法文档** (8 份)：感知算法、规划算法、控制算法的设计文档和技术报告
- **法规文档** (3 份)：GB/T 国家标准、ISO 26262 功能安全标准、测试规范
- **测试文档** (10 份)：测试场景库、测试用例、测试报告

**总文档数**: 26 份  
**总 Chunk 数**: 120+

### 2.2 算法查询

#### 示例 1：查询算法原理

**查询**：目标检测算法的原理是什么？

**系统响应**：

```
目标检测算法的核心原理如下 [1]：

**1. 特征提取**
- 使用卷积神经网络（CNN）提取图像特征
- 常用骨干网络：ResNet, VGG, MobileNet
- 提取多尺度特征以检测不同大小的目标

**2. 候选区域生成**
- 两阶段方法（如 Faster R-CNN）：先生成候选区域，再分类
- 单阶段方法（如 YOLO, SSD）：直接预测目标位置和类别

**3. 目标分类和定位**
- 分类：判断目标类别（车辆、行人、自行车等）
- 定位：预测目标的边界框（x, y, w, h）
- 使用非极大值抑制（NMS）去除重复检测

**4. 后处理**
- 过滤低置信度检测结果
- 跟踪目标以保持时序一致性
- 融合多帧结果以提高鲁棒性

**性能指标** [2]：
- 准确率（Precision）：> 95%
- 召回率（Recall）：> 90%
- 推理速度：30 FPS（实时性要求）

引用来源：

[2] 感知算法设计文档.pdf - 性能指标 (p.25)
```

#### 示例 2：查询算法实现

**查询**：路径规划算法有哪些？

**系统响应**：

```
常用的路径规划算法包括 [1]：

**1. 基于搜索的算法**
- **A* 算法**：启发式搜索，适用于静态环境
- **Dijkstra 算法**：最短路径搜索，计算量大
- **RRT（快速随机树）**：适用于高维空间，计算效率高

**2. 基于优化的算法**
- **二次规划（QP）**：平滑轨迹，满足约束条件
- **模型预测控制（MPC）**：考虑动力学约束，实时性好
- **梯度下降**：优化路径代价函数

**3. 基于学习的算法**
- **强化学习**：通过试错学习最优策略
- **模仿学习**：从人类驾驶数据中学习
- **端到端学习**：直接从感知到控制

**算法选择建议** [2]：
- 高速公路：A* + 二次规划（结构化道路）
- 城市道路：RRT + MPC（动态环境）
- 泊车场景：混合 A* + 优化（狭小空间）

引用来源：
[1] 规划算法设计文档.pdf - 算法分类 (p.5-12)
[2] 规划算法设计文档.pdf - 算法选型 (p.20)
```

### 2.3 法规查询

#### 示例 1：查询国家标准

**查询**：自动驾驶分级标准是什么？

**系统响应**：

```
根据 GB/T 40429-2021《汽车驾驶自动化分级》标准 [1]，自动驾驶分为 6 个等级：

**L0 - 应急辅助**
- 驾驶员完全控制车辆
- 系统仅提供警告和瞬时辅助

**L1 - 部分驾驶辅助**
- 系统在某些情况下辅助驾驶员
- 驾驶员需要持续监控
- 示例：自适应巡航控制（ACC）

**L2 - 组合驾驶辅助**
- 系统同时控制横向和纵向运动
- 驾驶员需要持续监控并随时接管
- 示例：车道保持 + 自适应巡航

**L3 - 有条件自动驾驶**
- 系统在特定条件下完全控制车辆
- 驾驶员需要准备接管
- 示例：高速公路自动驾驶

**L4 - 高度自动驾驶**
- 系统在特定区域完全控制车辆
- 无需驾驶员接管
- 示例：限定区域的无人驾驶

**L5 - 完全自动驾驶**
- 系统在所有条件下完全控制车辆
- 无需驾驶员
- 示例：完全无人驾驶

引用来源：
[1] GB_T自动驾驶分级标准.pdf - 分级定义 (p.5-10)
```

```
- 探测距离: 200m

- 功耗较大
- 体积较大

**适用场景:** 高级别自动驾驶（L4/L5），需要高精度环境感知

### 2. 毫米波雷达 (Radar) [3][4]

- 探测距离: 150m
- 分辨率: 1°

- 功耗小
- 穿透能力强
- 体积小

**缺点:**
- 分辨率低，无法识别物体类型
- 无法提供 3D 点云

**选型建议:**

- **最佳实践**: 多传感器融合，发挥各自优势，提高系统鲁棒性

**成本对比:**
- 激光雷达方案：$8000-$15000
- 毫米波雷达方案：$500-$2000
- 融合方案：$10000-$20000
```

```
## 算法方案对比

针对查询 **"基于规则的规划算法 vs 基于学习的规划算法"**，以下是详细对比：

### 1. 基于规则的规划算法 [1][2]

**核心思想:**
- 使用预定义的规则和逻辑进行决策
- 基于状态机和决策树

**核心思想:**
- 从数据中学习驾驶策略
- 使用深度学习、强化学习
- 端到端学习

- 泛化能力强，适应复杂场景
- 可以从大量数据中学习
- 不需要手工设计规则

- **泊车场景**: 推荐基于规则的算法（安全关键）

- 使用学习算法处理复杂场景
- 规则算法监督学习算法的输出
- 确保系统始终在安全边界内
```

#### 示例：多传感器融合方案

**查询**：多传感器融合方案有哪些？

**系统响应**：

```
## 多传感器融合方案

针对查询 **"多传感器融合方案有哪些？"**，以下是主流融合方案汇总：

### 融合传感器组合

1. **激光雷达 + 摄像头 + 毫米波雷达** [1][2][3]
   - 作用：激光雷达提供 3D 点云，摄像头提供视觉信息，毫米波雷达提供速度测量
   - 优势：互补性强，鲁棒性高，全天候工作
   - 应用：L4/L5 自动驾驶
   - 成本：$15000-$25000

2. **摄像头 + 毫米波雷达** [4][5]
   - 作用：摄像头识别物体类型，毫米波雷达测量距离和速度
   - 优势：成本可控，全天候，满足 L2/L3 需求
   - 应用：L2/L3 辅助驾驶
   - 成本：$2000-$5000

3. **激光雷达 + 摄像头** [6]
   - 作用：激光雷达提供深度信息，摄像头提供纹理和颜色
   - 优势：高精度感知，适合城市场景
   - 应用：城市自动驾驶
   - 成本：$10000-$18000

4. **激光雷达 + 毫米波雷达** [7]
   - 作用：激光雷达提供高精度点云，毫米波雷达提供速度和全天候能力
   - 优势：互补性好，适合高速场景
   - 应用：高速公路自动驾驶
   - 成本：$8000-$15000

### 融合策略

**数据层融合** [8]
- 在原始数据层面融合传感器数据
- 优点：信息损失最小，精度最高
- 缺点：计算复杂度高，实时性差
- 应用：离线建图、高精度定位

**特征层融合** [9]
- 提取各传感器特征后融合
- 优点：平衡性能和计算量，实时性好
- 缺点：需要特征对齐，设计复杂
- 应用：目标检测、跟踪

**决策层融合** [10]
- 各传感器独立决策后融合
- 优点：计算效率高，模块化，易于维护
- 缺点：信息损失较大，精度较低
- 应用：障碍物检测、碰撞预警

### 融合算法

**卡尔曼滤波** [11]
- 适用于线性系统
- 融合多传感器的测量值
- 提供最优估计

**粒子滤波** [12]
- 适用于非线性系统
- 处理多模态分布
- 鲁棒性强

**深度学习融合** [13]
- 使用神经网络自动学习融合策略
- 端到端训练
- 性能上限高

### 性能提升

根据文档 [14]，多传感器融合相比单一传感器可以：
- 提高目标检测准确率 15-25%
- 降低误检率 30-40%
- 提升恶劣天气下的鲁棒性 50%+
- 扩大感知范围 20-30%

### 工程实践建议

**传感器布局** [15]:
- 前向：1 个激光雷达 + 3 个摄像头 + 2 个毫米波雷达
- 侧向：4 个摄像头 + 4 个毫米波雷达
- 后向：1 个摄像头 + 2 个毫米波雷达

**时间同步** [16]:
- 所有传感器需要时间同步（误差 < 10ms）
- 使用 GPS 或 PTP 协议
- 记录每个传感器的时间戳

**标定** [17]:
- 内参标定：每个传感器独立标定
- 外参标定：传感器之间的相对位置和姿态
- 定期重新标定（每 3 个月）
```

## 3. 查询最佳实践

### 3.1 如何编写有效的查询

#### 1. 使用具体的关键词

**❌ 不好的查询**：传感器怎么样？

**✅ 好的查询**：激光雷达的探测距离是多少？

**原因**：具体的关键词（"激光雷达"、"探测距离"）帮助系统精准定位相关文档。

#### 2. 包含专业术语

**❌ 不好的查询**：车怎么知道前面有东西？

**✅ 好的查询**：目标检测算法的原理是什么？

**原因**：使用专业术语（"目标检测算法"）可以更准确地匹配技术文档。

#### 3. 明确查询意图

**❌ 不好的查询**：激光雷达

**✅ 好的查询**：如何进行激光雷达的外参标定？

**原因**：明确的问题（"如何标定"）帮助系统理解您的需求。

#### 4. 拆分复杂问题

**❌ 不好的查询**：激光雷达和摄像头的参数、标定方法、融合策略是什么？

**✅ 好的查询**：
- 查询 1：激光雷达的技术参数是什么？
- 查询 2：如何进行激光雷达的标定？
- 查询 3：激光雷达和摄像头的融合策略有哪些？

**原因**：拆分后的查询更聚焦，系统可以提供更详细的答案。

#### 1. 事实性查询

## 4. 引用格式说明

### 4.1 引用结构

系统提供的每个引用包含以下信息：

```
[引用编号] 文档名称 - 章节名称 (页码)
```

**示例**：
```
[1] 激光雷达规格书.pdf - 技术参数 (p.5)
[2] 激光雷达标定文档.pdf - 外参标定流程 (p.8-12)
[3] ISO_26262功能安全标准.pdf - ASIL 等级定义 (p.12-15)
```

### 4.2 引用元数据

每个引用还包含以下元数据（在 API 响应中）：

- **document_type**: 文档类型（sensor_doc, algorithm_doc, regulation_doc, test_doc）
- **relevance_score**: 相关性评分（0-1）
- **authority_score**: 权威性评分（0-1）
- **excerpt**: 关键内容摘录

**示例**：
```json
{
  "document_name": "激光雷达规格书.pdf",
  "document_type": "sensor_doc",
  "sensor_type": "LiDAR",
  "section": "技术参数",
  "page": 5,
  "relevance_score": 0.92,
  "authority_score": 0.6,
  "excerpt": "Velodyne VLS-128 激光雷达探测距离: 200m, 分辨率: 0.1°"
}
```

### 4.3 引用排序规则

引用按以下规则排序：

1. **相关性优先**：相关性评分高的引用排在前面
2. **权威性次之**：相关性相近时，权威性高的引用排在前面
3. **文档类型权威性**：regulation_doc > algorithm_doc > sensor_doc > test_doc

### 4.4 如何验证引用

1. **检查引用编号**：确保答案中的引用编号与引用列表对应
2. **阅读摘录内容**：检查摘录内容是否支持答案
3. **查看相关性评分**：高相关性（> 0.8）的引用更可靠
4. **追溯原始文档**：如需详细信息，可查阅原始文档的对应章节和页码

## 5. 边界限制说明

### 5.1 系统能做什么

✅ **系统可以**：
- 查询现有文档中的事实性信息
- 对比不同技术方案的优缺点
- 汇总多个文档的相关信息
- 提供详细的文档引用
- 解释技术原理和方法
- 引用法规标准和测试规范

### 5.2 系统不能做什么

❌ **系统不能**：
- 预测未来技术趋势
- 分析实时传感器数据
- 诊断当前系统故障
- 提供超出知识库范围的信息
- 生成新的技术方案（只能基于现有文档）
- 替代专业工程师的判断

### 5.3 预测性查询拒绝

**示例查询**：预测下一代激光雷达的发展趋势

**系统响应**：

```
不提供预测、趋势分析或未来技术判断。

**建议改为查询：**
- 当前激光雷达的技术原理和实现方法
- 已有的激光雷达技术方案和案例
- 相关的标准和测试规范
- 激光雷达的技术演进历史（基于已有文档）
```

### 5.4 实时诊断查询拒绝

**示例查询**：判断当前激光雷达是否故障

**系统响应**：
```
查询 **"判断当前激光雷达是否故障"** 涉及实时诊断请求。

自动驾驶知识检索系统当前只提供知识检索和文档引用，
不提供实时故障诊断或问题分析。

**建议改为查询：**
- 激光雷达常见故障类型和特征
- 激光雷达故障排查流程和方法
```

## 6. 常见问题（FAQ）

### 6.1 查询相关

**Q1: 为什么我的查询没有返回结果？**

A: 可能的原因：
1. 查询关键词不在知识库覆盖范围内
2. 查询过于宽泛或模糊
3. 使用了错误的术语或表述

**解决方案**：
- 使用更具体的关键词
- 检查知识库范围（使用范围查询接口）
- 尝试使用同义词或相关术语

**Q2: 为什么系统拒绝了我的查询？**

A: 系统会拒绝以下类型的查询：
1. 预测性查询（预测未来趋势）
2. 实时诊断查询（分析实时数据）
3. 低相关性查询（知识库中没有相关资料）

**解决方案**：
- 参考系统提供的替代建议
- 重新表述查询，聚焦于事实性信息

**Q3: 如何提高查询的准确性？**

A: 最佳实践：
1. 使用具体的关键词和专业术语
2. 明确查询意图（查询什么、为什么）
3. 拆分复杂问题为多个简单查询
4. 使用中英文混合查询（如需要）

### 6.2 引用相关

**Q4: 如何验证引用的准确性？**

A: 验证步骤：
1. 检查引用的相关性评分（> 0.8 更可靠）
2. 阅读引用的摘录内容
3. 查阅原始文档的对应章节和页码
4. 对比多个引用的信息

**Q5: 为什么有些答案没有引用？**

A: 可能的原因：
1. 系统拒绝了查询（边界控制）
2. 检索结果相关性过低
3. 系统错误

**解决方案**：
- 检查系统响应中的错误信息
- 重新表述查询
- 联系技术支持

### 6.3 性能相关

**Q6: 为什么查询响应时间较长？**

A: 可能的原因：
1. 查询复杂度高（对比查询、汇总查询）
2. 系统负载高
3. 网络延迟

**正常响应时间**：
- 简单查询：< 2 秒
- 复杂查询：< 4 秒

**解决方案**：
- 拆分复杂查询
- 避免高峰时段
- 检查网络连接

**Q7: 如何提高查询性能？**

A: 优化建议：
1. 使用客户端缓存
2. 批量处理多个查询
3. 使用异步请求
4. 避免重复查询

### 6.4 知识库相关

**Q8: 如何查看知识库包含哪些文档？**

A: 使用范围查询接口：
```python
scope = client.get_scope(collection="ad_knowledge_v01")
print(f"文档数量: {scope.document_count}")
print(f"文档类型: {scope.document_types}")
print(f"覆盖领域: {scope.coverage_areas}")
```

**Q9: 如何添加新文档到知识库？**

A: 使用摄取接口：
```python
result = client.ingest(
    file_paths=["/path/to/document.pdf"],
    collection="ad_knowledge_v01",
    document_type="sensor_doc",
    metadata={"sensor_type": "LiDAR"}
)
```

**Q10: 知识库多久更新一次？**

A: 
- 知识库支持增量更新
- 新文档摄取后立即可用
- 建议定期审查和更新文档（每季度）

## 7. 使用技巧

### 7.1 高效查询技巧

#### 1. 使用同义词

如果一个查询没有返回满意的结果，尝试使用同义词：

- "激光雷达" ↔ "LiDAR"
- "摄像头" ↔ "Camera"
- "感知" ↔ "Perception"
- "规划" ↔ "Planning"

#### 2. 逐步细化查询

从宽泛到具体：

1. 第一次查询：激光雷达的技术参数
2. 第二次查询：Velodyne VLS-128 激光雷达的探测距离
3. 第三次查询：VLS-128 在雨雾天气下的探测距离

#### 3. 利用对比查询

当需要选型时，使用对比查询：

- 激光雷达 vs 毫米波雷达
- A* 算法 vs RRT 算法
- L3 vs L4 自动驾驶

#### 4. 利用多文档推理

当需要全面了解时，使用汇总查询：

- 多传感器融合方案有哪些？
- 感知算法有哪些类型？
- 自动驾驶测试场景有哪些？

### 7.2 引用使用技巧

#### 1. 优先查看高相关性引用

相关性评分 > 0.8 的引用更可靠，优先阅读。

#### 2. 交叉验证多个引用

如果多个引用支持同一结论，可信度更高。

#### 3. 关注权威文档

法规文档（regulation_doc）的权威性最高，其次是算法文档（algorithm_doc）。

#### 4. 追溯原始文档

如需详细信息，根据引用的章节和页码查阅原始文档。

### 7.3 避免常见错误

#### ❌ 错误 1：查询过于宽泛

**不好的查询**：自动驾驶
